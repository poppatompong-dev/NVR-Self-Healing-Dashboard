# Avigilon NTP Sync & Reporting Tools

A robust set of scripts designed to manage, synchronize, and report on Avigilon NVR and camera times across a closed local network.

## The Problem
In a highly secured, closed-circuit network (CCTV) without internet access, cameras often drift in time. Without a central public NTP server, footage across different cameras can become unsynchronized, which is detrimental to legal or investigative review. 

Additionally, some camera models (like Hikvision, Dahua, or ONVIF generic cameras) handle time offsets, Time Zones, and Daylight Saving Time (DST) differently, causing time jumps when relying purely on the NVR's built-in NTP.

## The Solution
This project establishes a **Hierarchical NTP Engine** coupled with an aggressive **SOAP/ONVIF Watchdog**.

1. **Master NVR (NVR 1)** acts as the local stratum server for the entire network.
2. **Client NVR (NVR 2)** and all cameras sync to the Master NVR.
3. **NVR Watchdog** runs every 15 minutes to actively poll all 33 cameras via ONVIF SOAP requests. If a camera drifts by more than 2 seconds, it aggressively forces a manual time sync, then drops it back into NTP mode.
4. **Unified Reporting** runs twice a day, generating a beautiful retro-CLI style HTML report and emailing it to administrators (if a temporary external network gateway is available, or sent to a local mail relay).

## Features
* **Dual-NVR Support:** Handles both 10.0.3.138 (Master) and 10.0.3.137 (Client).
* **Parallel Processing:** Uses `ThreadPoolExecutor` to scan 33 cameras in seconds rather than minutes.
* **Flapping Detection:** Silently logs cameras that bounce offline/online to prevent email spam, summarizing the events in the evening report.
* **Retro-Terminal Report:** Emails are formatted as a clean, hacker-style console interface for quick scannability.

## Installation

1. Copy this entire folder to `D:\avigilon_ntp_tools` on the Master NVR.
2. Ensure Python 3.x is installed and added to `PATH`.
3. Open **PowerShell as Administrator**.
4. Run the installation script:
   ```powershell
   cd D:\avigilon_ntp_tools\scripts
   .\install_tasks.ps1
   ```
5. The script will automatically install dependencies (`requests`) and set up Windows Task Scheduler to run the watchdog (every 15 mins) and the email report (8:00 AM & 5:00 PM).

## Directory Structure
* `src/` - Core production scripts (`nvr_watchdog.py`, `unified_report.py`)
* `scripts/` - Installation and setup PowerShell scripts
* `data/` - Runtime data (camera inventory, status state, downtime logs)
* `docs/` - Documentation and architecture diagrams
* `logs/` - Historical HTML reports generated by the system
* `experiments/` - Historical brute-force and testing scripts used during development
